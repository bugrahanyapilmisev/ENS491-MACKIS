Related Instruction
Hizmet Uygulamalarını Geliştirme Yönergesi (IIT-S120-01)
Aim
To ensure the execution of application development activities required at Sabancı University.
Scope
Planning, design, and application activities carried out for application development purposes.
Definitions / Abbreviations

IT: Information Technologies

IT Manager: IT Manager responsible for the relevant system/service

Unit Manager: Manager of the requesting unit

Application Manager: IT employee responsible for the relevant system/service

Developer: University or outsourced employee who developed the application

Application Team: Unit employees who carry out the work

Document that plans resources, time, assignments, etc.

 

Implementation Steps

 Responsibility

Outputs/Timing

1. Application Request Acceptance and Scope Determination

    

1.1. Application requests are evaluated and received within the scope of the Requirements Gathering and Assessment (PIT-S110-0101) procedure.

 IT Manager

Requirement accepted for application and resource planning completed

1.2. The IT manager establishes the IT application team that will participate in the work and determines the application project manager. 

 IT Manager /On the planned date

1.3. The IT manager informs the relevant unit manager.

2. Planning

    

2.1. The application manager creates the Application Plan Document, considering the academic calendar, the application team's workload, and process work plans.

Application Manager

Application Plan Document/

2.2. The application manager shares the plan document with the application team, and the Application Plan Document is updated based on feedback.

 

 Application Manager Application Plan Document/

2.3. The final version of the application plan document is notified to the IT and relevant unit managers, and work begins.

 

 Application Manager Application Plan Document/

3. Analysis, Design, Development, and Control

    

3.1. During the analysis, design, development, and control steps, the application manager and the application team carry out the work within the timeframe specified in the Application Plan Document.

  / On the planned date

3.2. During the work, the Application Plan Document is updated in cases that may affect the process and outputs. The new situation is communicated to IT and unit managers.
